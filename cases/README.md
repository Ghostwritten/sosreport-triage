# Case Library

These cases are meant for replay and calibration, not copy-paste diagnosis.

Use them to:

- compare a live customer case against known patterns
- train new support engineers on evidence-based reasoning
- test whether the skill produces consistent judgments

## Suggested Structure

- `templates/case-template.md`: create new cases with a consistent schema
- pattern cases: representative resource incidents
- future `fixtures/`: optional sanitized path snippets or extracted file fragments

## Minimum Fields For Each Case

- scenario
- observed facts
- likely impact
- most likely explanations
- evidence gaps
- common misreads
- expected severity and confidence
